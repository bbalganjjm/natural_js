마. 포맷 룰(Format rules)

룰 명칭은 대소문자를 가리지 않습니다.

룰명 설명
["commas"] 숫자 3 자리마다 콤마(,) 를 추가 해 줍니다. 소수점이 있어도 소수점 앞부분만 처리 합니다.
["rrn"] 주민등록번호(Resident registration number) 형식으로 변경
["ssn"]

문자열 길이에 따라서 주민등록번호(Resident registration number) 형식(13자리)이나 미국의 사회보장번호(Social Security Number) 형식()으로 변경

나중에 미국 사회보장번호 체크로직만 남겨 놓고 주민등록번호 체크 로직을 제거 할 예정이니 주민등록번호는 "rrn" 을 사용 해 주세요.

["cno"]

Deprecated. "kbrn" 룰로 변경(2017.09.26)

 사업자등록번호 형식으로 변경
["kbrn"] 한국 사업자등록번호 형식으로 변경
["kcn"] 한국 법인번호 형식으로 변경
["upper"] 대문자로 변경
["lower"] 소문자로 변경
["capitalize"] 첫번째 영문자를 대문자로 변경
["zipcode"] 우편번호 형식으로 변경

["phoneNum"]

Deprecated. phone 룰로 변경(2017.07.27)

 전화번호 형식으로 변경
["phone"] 전화번호 형식으로 변경
["realnum"]

의미없는 0 제거

ex)

0100.0 -> 100

0100.10 -> 100.1
["trimToEmpty"] trim 후 값이 없으면 공백으로 표시
["trimToZero"] trim 후 값이 없으면 0 으로 표시
["trimToVal", "valStr"] trim 후 값이 없으면 valStr 값으로 표시
["date", 4|6|8|10|12|14, "month"|"date", { "datepicker options" : "" }]

날짜 형식으로 표시

기본 날짜형식은 N.context.attr("data").formatter.date 에서 설정 가능 합니다.

1. 다음은 두 번째 인자 값들에 대한 설명 입니다.

대쉬(-), 콜론(:) 등의 날짜와 시간을 구분하는 문자는 Config(N.config, natural.config.js 파일) 의 N.context.attr("data").formatter.date 오브젝트의 함수들로 정의 되어 있습니다. 날짜/시간 구분 문자는 해당 위치에서 변경 가능 합니다.

  숫자 타입으로 길이를 지정하면 다음과 같이 변환 됩니다.

  • 4:년
  • 6:년-월
  • 8:년-월-일
  • 10:년-월-일 시
  • 12:년-월-일 시:분
  • 14:년-월-일 시:분:초

  문자 타입으로 데이트 포멧 룰을 지정하면 다음과 같이 변환 됩니다.

  • Y : 년(4자리)
  • y : 년(2자리)
  • m : 월
  • d : 일
  • H : 시
  • i : 분
  • s : 초
  • 예) "1999/12/31" : "Y/m/d"
    예) "99/12/31" : "y/m/d"
    예) "31/12" : "d/m"
    예) "12/31/1999" : "m/d/Y"
    예) "1999-12-31 12:01:59" : "Y-m-d H:i:s"

2. 다음은 세 번째 인자 값들에 대한 설명 입니다.

  • date : Datepicker 표시
  • month : Monthpicker 표시

3. 네 번째 인자는 Datepicker 컴포넌트의 옵션들을 JSON String 형태로 넣으면 옵션이 적용 됩니다.
["time"]

시간 형식으로 표시
["limit, cutLength, "replaceStr"] 문자열을 cutLength 만큼 자르고 replaceStr로 치환
["replace", "target string", "replace string"] "target string" 을 "replace string"로 치환
["lpad", length, "fill string"] length만큼 왼쪽부터 "fill string" 으로 채움
["rpad", length, "fill string"] length만큼 오른쪽부터 "fill string" 으로 채움
["mask", "phone", "masking character"]

전화번호를 마스킹 합니다.

"masking character" 인자를 입력하지 않으면 "*" 문자로 치환 됩니다.

["mask", "email", "masking character"]

이메일 주소를 마스킹 합니다.

"masking character" 인자를 입력하지 않으면 "*" 문자로 치환 됩니다.

["mask", "address", "masking character"]

주소를 마스킹 합니다.

"masking character" 인자를 입력하지 않으면 "*" 문자로 치환 됩니다.

["mask", "name", "masking character"]

이름을 마스킹 합니다.

"masking character" 인자를 입력하지 않으면 "*" 문자로 치환 됩니다.

["mask", "rrn", "masking character"]

주민등록 번호를 마스킹 합니다.

"masking character" 인자를 입력하지 않으면 "*" 문자로 치환 됩니다.

["generic", "사용자서식"]

사용자 서식을 이용하여 문자열을 포멧 합니다.

generic 과 numeric 은 Mask JavaScript API (v0.4b) 을 기본으로 하여 Date 마스킹 부분은 제거하고 한글인식 부분과 0보다 작은 값들(수수점 뒤 값들)에 대한 처리를 추가하고 * 문자와 공백을 허용(기존 * 문자는 ~ 문자로 변경) 하도록 변경 하였습니다. 자세한 사용법과 예는 위 링크를 참조 하세요.

  • # : 숫자, 공백문자
  • @ : 한글(자음/모음), 영문자, 공백문자
  • ~ : 한글(자음/모음), 영문자, 숫자, 공백문자

["numeric" "사용자서식", "option"]

사용자 서식을 이용하여 숫자 문자열을 포멧 합니다.

option 인자 값에 따라 소숫점을 다음과 같이 처리 합니다.

  • ceil : 무조건올림 (소수점 처리가능)
  • floor : 무조건버림(소수점 처리가능)
  • round : 반올림(소수점 처리가능)

generic 과 numeric 은 Mask JavaScript API (v0.4b) 을 기본으로 하여 Date 마스킹 부분은 제거하고 한글인식 부분과 0보다 작은 값들(수수점 뒤 값들)에 대한 처리를 추가하고 * 문자와 공백을 허용(기존 * 문자는 ~ 문자로 변경) 하도록 변경 하였습니다. 자세한 사용법과 예는 위 링크를 참조 하세요.
