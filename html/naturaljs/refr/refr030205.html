예제 Examples

1. N.form이나 N.grid로 데이터를 바인딩하기 전에, 컬럼 값이 바인딩될 요소의 data-validate 속성에 유효성 검증 규칙을 선언하기.

1. Before binding data with N.form or N.grid, declare validation rules in the data-validate attribute of the element where the column value will be bound.

<input id="col01" type="text" data-validate='[["required"]]'>

위와 같이 입력 요소에 선언하면, 포커스가 해당 요소에서 벗어날 때 유효성 검증이 실행되며, 유효성 검증에 실패할 경우 실패 원인을 입력 요소 옆에 툴팁으로 보여 줍니다.

When declared on an input element as described above, validation is executed when the focus leaves the element. If validation fails, the reason for the failure is displayed as a tooltip next to the input element.

2. N.validator 라이브러리를 사용하여 유효성 검증하기.

2. Perform validation using the N.validator library.

입력 요소에 data-validate='[["rangevalue", 1, 9]]' 와 같이 검증 규칙을 선언하면 프레임워크 내부에서는 "rangevalue" 다음의 1과 9를 [1, 9]와 같이 Array 형태로 변환하여 N.validator.rangevalue 함수의 인수로 실행합니다.

When you declare a validation rule like data-validate='[["rangevalue", 1, 9]]' on an input element, the framework internally converts the numbers following "rangevalue" into an array, [1, 9]. This array is then used as arguments for the N.validator.rangevalue function to execute the validation logic.

유효성 검증 규칙 함수를 명령으로 처리하는 방법은 다음과 같습니다.

The method for handling validation rule functions as commands is as follows:

// N.validator.{rule}("String to validate", ...arguments);

N.validator.rangevalue("Validation string", [1, 9]); // false

3. 요소에 바인딩된 Validator 관련 이벤트를 직접 실행하기.

3. Directly triggering Validator-related events bound to an element.

Validator는 룰 대신 입력 요소를 지정하면, 해당 입력 요소에 "validate" 이벤트를 생성합니다. 이 이벤트는 jQuery의 trigger 메서드를 통해 직접 실행할 수 있습니다.

When you specify an input element instead of a rule for the Validator, it generates a "validate" event on that input element. This event can be directly triggered using jQuery's trigger method.

N("input[name='targetEle']").trigger("validate");

4. 동적으로 유효성 검증 규칙을 변경하고 즉시 반영하기.

4. Dynamically change and immediately apply validation rules.

N("selector").data("validate", [["required"]]).trigger("validate");

위 코드가 작동하기 위해서는 컴포넌트로 데이터를 바인딩하기 전에 대상 요소에 data-validate 속성으로 유효성 검증 규칙이 1개 이상 선언되어 있어야 합니다.

To ensure the code functions properly, at least one validation rule must be declared using the data-validate attribute on the target element before binding data to the component.

전체 유효성 검증 룰에 대한 설명은 [유효성 검증 룰 목록] 탭을 참고하세요.

Refer to the [Validation rule list] tab for a full description of the format rules.
