Name Type Default Required Description
data array[json object] undefined O

N.form에 바인딩할 데이터를 지정합니다.

Specifies the data to bind to N.form.

N.form은 단일 데이터를 표현하는 컴포넌트이지만 바인딩되는 데이터는 array[json object] 타입입니다.

N.form is a component that expresses single data, but the data to be bound is of type array[json object].
row number -1 X

data옵션으로 지정한 목록 데이터에서 폼에 바인딩할 행의 index 값을 입력합니다.

Set the index value of the row to bind to the form from the list data specified with the data option.

기본값은 –1이지만 값을 입력하지 않으면 0으로 세팅되어 목록 데이터의 첫 번째 행 데이터를 바인딩합니다.

The default value is -1, but if not set, it is set to 0 to bind the first row data of the list data.
context jQuery object|jquery selector string null O

N.form을 적용할 context 요소를 지정합니다.

Specifies the context element to apply N.form.

N.form의 context 요소는 table, div, section 등 영역을 표현하는 태그로 작성하면 됩니다.

The context element of N.form can be written with tags that express areas such as table, div, section and etc.

데이터의 프로퍼티명과 요소의 id 속성 값이 같으면 데이터가 바인딩됩니다.

If the property name of the data and the value of the id attribute of the element are the same, the data is bound.

Natural-JS의 모든 데이터 관련 컴포넌트는 빠른 바인딩 속도를 위해 어쩔 수 없이 id 속성을 사용합니다.

All data-related components of Natural-JS inevitably use id attribute for fast binding speed.

validate boolean true X

false로 설정하면 입력 요소에서 focus-out 되었을 때 입력 값에 대한 유효성 검증을 실행하지 않습니다.

If set to false, validation is not performed on the input value when the input element is focus-out.
html boolean false X

true로 설정하면 data를 bind 할 때 데이터의 HTML이 적용됩니다.

If set to true, HTML of data is applied when binding data.
revert boolean false X

true로 설정하면 revert 기능이 활성화되고 revert 메서드를 사용할 수 있습니다.

If set to true, the revert function is enabled and the revert method can be used.

revert 기능이 활성화되면 최초 바인딩된 데이터가 메모리에 적제되므로 메모리 사용량이 증가합니다.

When the revert function is activated, the memory usage increases because the first bound data is loaded into the memory.
unbind boolean true X

true로 설정하면 unbind 기능이 활성화되고 unbind 메서드를 사용할 수 있습니다.

If set to true, the unbind function is enabled and the unbind method can be used.

unbind 기능이 활성화되면 최초 바인딩된 데이터가 메모리에 적제되므로 메모리 사용량이 증가합니다.

When the unbind function is activated, the memory usage increases because the first bound data is loaded into the memory.

폼의 context 요소는 다시 생성되지 않기 때문에 원칙적으로 add 메서드와 bind 메서드는 용도에 따라 구분해서 사용해야 합니다. 이미 데이터가 바인딩된 폼 요소에 unbind 메서드를 호출해도 완벽하게 이전 상태로 되돌릴 수는 없습니다. 어쩔 수 없는 경우가 아니면 사용을 권장하지 않습니다.

Since the form's context element is not re-created, add and bind methods should be used separately according to the usage. Calling the unbind method on a form element to which data has already been bound does not completely revert to the previous state. We do not recommend using it unless you have no choice.
autoUnbind boolean false X

true로 설정하면 폼에 데이터를 바인딩하거나 생성 한 다음 같은 폼 요소에 다시 데이터를 바인딩하기 전에 unbind 메서드를 자동으로 호출해 주어서 폼 요소를 재 활용할 수 있게 해 줍니다.

If set to true, form elements can be reused by automatically calling the unbind method before rebinding the data to the same form element after binding or creating data from the form.

true로 설정하면 add 메서드를 호출했을 때 로우 데이터가 데이터 목록의 맨 앞에 추가됩니다.

If set to true, row data created to the top of the data list when the add method is called.
tpBind boolean false X

true로 설정하면 컴포넌트 초기화 전에 입력 요소에 바인딩되어 있는 이벤트와 format, validate, dataSync 등의 컴포넌트 이벤트의 충돌을 방지해 줍니다.

If set to true, it prevents the conflict of the event bound to the input element before component initialization and the component event such as format, validate, and dataSync.

format이 정상적으로 동작하지 않거나 바인딩되어 있는 데이터가 의도한 대로 핸들링되지 않을 때, 알 수 없는 오류가 발생했을 때 true로 설정 바랍니다.

Set to true when an unknown error occurs, such as the format not working properly or the bound data is not handled as intended.

대상 요소의 data-format 속성이 아닌 객체 타입으로 format 룰을 지정합니다.

옵션 오브젝트는 Formatter 메뉴의 [ 생성자 ] 탭의 rules 인자의 설명을 참고해서 작성 바랍니다.

Create an option object by referring to the description of the rules argument in the [ Constructor ] tab of the Formatter menu.

Specify the validation rule with the object type, not the data-validate attribute of the target element.

onBeforeBindValue function null X

데이터 프로퍼티 값이 요소에 바인딩되기 전에 실행되는 이벤트 핸들러를 정의합니다.

Defines an event handler that executes before a data property value is bound to an element.

Config(natural.config.js)에 전역 옵션으로 설정할 경우 N.form을 사용하는 모든 컴포넌트(N.grid, N.list)들에서도 적용됩니다. onBeforeBindValue 이벤트를 사용하면 N.form, N.grid, N.list로 바인딩되는 모든 값들과 요소들을 한 번에 핸들링할 수 있습니다.

If you set it as a global option in Config(natural.config.js), all components using N.form(N.grid, N.list) also apply. Using the onBeforeBindValue event, you can handle all column values ​​and elements bound to N.form, N.grid, and N.list at once.

onBeforeBindValue 이벤트를 N.config에 전역 이벤트 옵션으로 설정하면 컴포넌트 초기화 시 정의 한 onBeforeBindValue 이벤트 핸들러의 결괏값이 전역 이벤트 핸들러의 val(arguments[1]) 인자로 지정됩니다. 때문에 다른 이벤트들처럼 전역으로 설정한 이벤트 핸들러를 컴포넌트 이벤트에서 중단할 수 없습니다.

If the onBeforeBindValue event is set as a global event option in N.config, the result value of the onBeforeBindValue event handler defined at component initialization is specified as the val (arguments [1]) parameter of the global event handler. Therefore, you cannot stop the globally set event handler from the component event like other events.

onBeforeBindValue : function(ele, val, action) {
    // this : N.form 인스턴스
    // ele : 바인딩될 요소
    // val : 바인딩될 값
    // action : N.form 인스턴스의 메서드 명 - "bind" | "val"
    return val; // 반드시 처리된 값을 다시 리턴해야 합니다.
}
onBeforeBindValue : function(ele, val, action) {
    // this : N.form instance
    // ele : The element to be bound
    // val : The value to be bound
    // action : Method name of N.form instance - "bind" | "val"
    return val; // The processed value must be returned again.
}
