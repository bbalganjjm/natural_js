jQuery selector 확장 jQuery selector extensions

Natural-CORE 에는 CSS1~3에서 지원되지 않는 형태의 패턴 검색을 위해 요소의 속성 값을 정규 표현식으로 평가할 수 있는 regexp 필터 셀렉터가 확장되어 있습니다. 이를 통해 보다 복잡한 형태의 요소 검색을 지원합니다.

Natural-CORE extends the regexp filter selector that can evaluate element attribute values as regular expressions to search for patterns that are not supported in CSS1~3. This supports more complex element search.

[selector]:regexp( attributeName[:propertyName], expr ) Return: jQuery Object
  • selector
    Type : string

    기본 selector 문자열입니다.

    Default selector string.

    생략할 경우 모든 요소에 대해 평가가 이뤄지기 때문에 가능하면 기본 selector를 지정하는 것이 좋습니다.

    If omitted, it is recommended to specify the default selector if possible, because all elements are evaluated.

  • attributeName
    Type : string

    평가하려는 값을 갖고 있는 HTML DOM 요소의 속성 명입니다.

    The attribute name of the HTML DOM element whose value you want to evaluate.

    아래에 나열된 속성을 제외한 모든 속성 값은 .attr(attributeName) 함수의 반환 값으로 평가됩니다.

    All property values except those listed below are evaluated as the return value of the .attr(attributeName) function.

    • data : 요소의 data속성을 추출합니다. Extract the element's data attribute. 반드시 propertyName으로 하위 속성을 지정해야 합니다. You must specify a child property with propertyName. .data(properyName) 함수의 반환 값과 같습니다. Equivalent to the return value of the .data(properyName) function.
    • css : 요소의 css 속성을 추출합니다. Extract the element's css attribute. 반드시 propertyName으로 하위 속성을 지정해야 합니다. You must specify a child property with propertyName. .css(propertyName) 함수의 반환 값과 같습니다. Equivalent to the return value of the .css(propertyName) function.
    • class : 요소의 class 속성을 추출합니다. Extract the element's class attribute. class 속성 값이 둘 이상인 요소의 경우 각 클래스에 대해 평가가 수행됩니다. For elements with more than one class attribute value, evaluation is performed for each class.
  • propertyName
    Type : string
    attributeName이 data 혹은 css인 경우 하위 속성을 의미합니다. If attributeName is data or css, it means sub-attribute. 이외의 경우 값을 지정해도 무시됩니다. In other cases, specifying a value is ignored.
  • expr
    Type : string

    정규 표현식 문자열입니다. 큰따옴표나 작은따옴표로 감싸지 않아야 합니다.

    Regular expression string. It should not be enclosed in double or single quotation marks.

    /* href 속성에 "Mr.Lee" 혹은 "Mr.Kim"을 포함하는 모든 a 태그 추출 *//* Extract all a tags that contain "Mr.Lee" or "Mr.Kim" in the href attribute. */
N("a:regexp(href, 'Mr\\.(Lee|Kim)')"); // (X)
N('a:regexp(href, "Mr\\.(Lee|Kim)")'); // (X)
N("a:regexp(href, Mr\\.(Lee|Kim))"); // (O)

:regexp 필터 속성 및 예제 :regexp filter attribute and examples

1. 다중 class 속성 값을 가진 요소 필터링

1. Filtering elements with multiple class attribute values

<div class="someClassA classB"></div>

N("div:regexp(class, ^someClass)")
<div class="searchBtn"></div>
<div class="searchBtn"></div>
<div class="searchButton"></div>

N("div:regexp(class, Btn$|Button$)")

2. data속성의 프로퍼티 명으로 필터링

2. Filter by property name of data attribute

<div data-sample="test-1"></div>
<div data-sample="test-2"></div>

N("div:regexp(data:sample, test-[1-2])")

3. css 속성의 프로퍼티 명으로 필터링

3. Filter by property name of css attribute

<div style="width:128px;"></div>

N("div:regexp(css:width, ^128px)")

4. id 속성 값으로 필터링

4. Filter by id attribute value

<div id="page-1"><div id="page-2"><div id="page-3">

N("div:regexp(id, page-[0-9])")
