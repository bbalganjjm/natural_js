검증 룰 목록 Validation rule list

룰 명은 대문자와 소문자를 다르게 인식하지 않습니다.

Rule names do not recognize uppercase and lowercase letters differently.

룰명

설명

Rule name

Description
["required"]

필수입력

Required input
["alphabet"]

영문자만 입력 가능

Only English characters
["integer"]

숫자(정수)만 입력 가능

Only numbers(integer)
["korean"]

한글만 입력 가능

Korean only
["alphabet+integer"]

영문자와 숫자(정수)만 입력 가능(+로 구분하고 순서는 관계없음)

Only alphanumeric(integers)(separated by +, regardless of order)
["integer+korean"]

숫자(정수)와 한글만 입력 가능(+로 구분하고 순서는 관계없음)

Only numbers(integer) and Hangul(separated by +, regardless of order)
["alphabet+korean"]

영문자와 한글만 입력 가능(+로 구분하고 순서는 관계없음)

Only English and Korean characters(separated by +, regardless of order)
["alphabet+integer+korean"]

영문자, 숫자(정수), 한글만 입력 가능(+로 구분하고 순서는 관계없음)

Only English, numbers(integer), and Hangul(separated by +, regardless of order)
["integer+dash"]

숫자(정수), 대시(-)만 입력 가능(+로 구분하고 순서는 관계없음)

Only numbers(integer) and dash(-)(separated by +, regardless of order)
["integer+commas"]

숫자(정수), 콤마(,)만 입력 가능(+로 구분하고 순서는 관계없음)

Only numbers(integer) and commas(,)(separated by +, regardless of order)
["number"]

숫자(정수), 콤마(,), 점(.)만 입력 가능

Only numbers(integer), comma(,), and dot(.)

["decimal", length]

(유한)소수 점 length 번째 자리까지 입력 가능

You can enter up to the "length" digit of the decimal(Limited) point
["phone", isPartialDigits]

전화번호 형식에 맞는지 검사

Check South Korean phone number format.

isPartialDigits 이 true 이면 전화번호 마지막 4자리중 1자리만 입력되어도 통과 됩니다.

If isPartialDigits is true, it will pass even if only 1 of the last 4 digits of the phone number is entered.
["email"]

e-mail 형식에 맞는지 검사

Checks if it matches the e-mail format
["url"]

URL 형식에 맞는지 검사

Checks for URL format
["zipcode"]

우편번호 형식에 맞는지 검사

Checks if it is in the format of the postal code in Korea
["rrn"]

주민등록번호 형식과 일치하는지 검사

Checks if it matches the Korean Resident Registration Number format
["ssn"]

미국 사회보장번호 형식에 맞는지 검사

Checks if it matches the US Social Security Number format
["frn"]

외국인등록번호 형식과 일치하는지 검사

Checks if it matches the format of the foreigner registration number in Korea
["frn_rrn"]

외국인등록번호 또는 주민등록번호 형식과 일치하는지 검사

Checks if it matches the foreigner registration number or resident registration number format
["kbrn"]

사업자등록번호 형식과 일치하는지 검사

Checks if it matches the Korean business registration number format
["kcn"]

법인번호 형식과 일치하는지 검사

Checks if it matches the Korean corporate number format
["date"]

날짜 형식과 일치하는지 검사

Checks if it matches the date format
["time"]

시간 형식과 일치하는지 검사

Checks if it matches the time format
["accept", "word"]

"word" 값만 입력 가능

Only "word" value
["notAccept", "word""]

"word" 값만 입력 불가능

Only not "word" value
["match", "word"]

"word" 값이 포함된 값만 입력 가능

Only values that contain the "word" value
["notMatch", "word"]

"word"가 포함된 값은 입력 불가능

Only values that not contain the "word" value
["acceptFileExt", "fileExtention"]

"fileExtention" 값이 포함된 확장자만 입력 가능

Only extensions with "fileExtention" value
["notAcceptFileExt", "fileExtention"]

"fileExtention"가 포함된 확장자는 입력 불가능

Only extensions without "fileExtention" value
["equalTo", "input"]

"input"의 값과 같아야 입력 가능

Input is possible only when the value of "input" is the same.
["maxlength", length]

length 이하의 글자 수만 입력 가능

Only the number of characters less than "length"
["minlength", length]

length 이상의 글자 수만 입력 가능

Only the number of characters longer than “length”
["rangelength", startLength, endLength]

startLength 글자 수에서 endLength 글자 수까지만 입력 가능

Only between the number of characters from startLength and the number of endLength characters
["maxbyte", byteLength, defaultCharByteLength]

byteLength 바이트 이하만 입력 가능

Only byteLength bytes or less

defaultCharByteLength는 영문, 숫자, 기본 특수문자 등을 제외한 한글, 한글 특수 문자 등의 기본 바이트 길이입니다. 입력하지 않으면 Config(natural.config.js)의 N.context.attr("core").charByteLength 값이 적용됩니다.

defaultCharByteLength is the default byte length for Hangul, Hangul special characters, etc., excluding English, numbers, and basic special characters. If not entered, the value of N.context.attr("core").charByteLength of Config(natural.config.js) is applied.
["minbyte", byteLength, defaultCharByteLength]

byteLength 바이트 이상만 입력 가능

Only byteLength bytes or more

defaultCharByteLength는 영문, 숫자, 기본 특수문자 등을 제외한 한글, 한글 특수 문자 등의 기본 바이트 길이입니다. 입력하지 않으면 Config(natural.config.js)의 N.context.attr("core").charByteLength 값이 적용됩니다.

defaultCharByteLength is the default byte length for Hangul, Hangul special characters, etc., excluding English, numbers, and basic special characters. If not entered, the value of N.context.attr("core").charByteLength of Config(natural.config.js) is applied.
["rangebyte", startByteLength, endByteLength, defaultCharByteLength]

startByteLength 바이트에서 endByteLength 바이트 까지만 입력 가능

Only between startByteLength "stratValue" and endByteLength bytes

defaultCharByteLength는 영문, 숫자, 기본 특수문자 등을 제외한 한글, 한글 특수 문자 등의 기본 바이트 길이입니다. 입력하지 않으면 Config(natural.config.js)의 N.context.attr("core").charByteLength 값이 적용됩니다.

defaultCharByteLength is the default byte length for Hangul, Hangul special characters, etc., excluding English, numbers, and basic special characters. If not entered, the value of N.context.attr("core").charByteLength of Config(natural.config.js) is applied.
["maxvalue", number]

"value" 이하의 값만 입력 가능

Only "value" values or less
["minvalue", number]

"value" 이상의 값만 입력 가능

Only "value" values or more
["rangevalue", startNumber, endNumber]

"startValue" 값에서 "endValue" 값 까지만 입력 가능

Only between "startValue" and "endValue" values
["regexp", "regexp string", "flag", "message"]

입력한 "regexp string"의 조건으로 유효성 검증 후 검증에 실패하면 "message" 값을 툴팁으로 표시합니다.

If validation fails after validation with the condition of the entered "regexp string", the value of "message" is displayed as a tooltip.

정규식 평가는 다음과 같이 실행됩니다.

Regular expression evaluation is executed as follows :

\{regexp string}\{flag}

"flag"가 없으면 빈 값으로 넣어주세요.

If there is no "flag", enter it as an empty value.

"message"를 입력하지 않으면 기본 메시지("필드 검증에 통과하지 못했습니다.")가 표시됩니다.

If "message" is not entered, the default message("It Can't pass the field verification.") is displayed.
