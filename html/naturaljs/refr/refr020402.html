기본 옵션 Default Options

Name Type Default Required Description
append boolean false X

true 로 설정하면 로드된 페이지를 덮어쓰지 않고 target 옵션으로 지정한 요소에 추가합니다.

If set to `true`, the loaded page will be appended to the element specified by the `target` option rather than overwritten.
urlSync boolean true X

false 로 설정하면 서버로 요청할 때의 location.href 와 서버에서 응답을 받았을 때의 location.href 가 다르더라도 응답을 차단하지 않습니다.

If set to false, the response will not be blocked even if location.href when requesting to the server and location.href when receiving a response from the server are different.

알 수 없는 이유로 서버 응답이 차단되었을 경우 이 옵션을 false 로 설정해서 테스트해 보세요.

If the server response is blocked for unknown reasons, set this option to false to test.
dataIsArray boolean false X

true 로 설정하면 N().comm 에서 N 함수의 인수로 지정한 파라미터 객체를 array 타입으로 지정할 수 있습니다.

If set to true, the parameter object specified as an argument of N function in N().comm can be specified as an array type.

Communicator 를 N(params).comm(url).submit();와 같이 사용할 때 params 의 객체 타입이 array 일 때 dataIsArray 옵션이 false 로 지정되어 있다면 array 의 첫 번째 오브젝트만 전송됩니다. 이 문제의 원인은 jQuery 함수의 인수를 array(jQuery([{}])) 또는 object($({}))로 설정한 후에 get 함수를 호출하면 둘 다 array([{}])를 반환하기 때문입니다. 불편하더라도 array 를 서버에 전송할 때는 dataIsArray 를 true 로 설정하거나 object 에 array 를 담아서 사용 바랍니다.

If Communicator is used as N(params).comm(url).submit();, only the first object in the array is sent when the object type of params is array and the dataIsArray option is set to false. The cause of this problem is that if you call get function after setting the argument of jQuery function to array(jQuery([{}])) or object($({})), both return array ([{}]). Even if inconvenient, when sending an array to the server, set dataIsArray to true or use an object in an array.

Communicator 를 N.comm(params, url).submit();와 같이 사용하면 dataIsArray 옵션을 true 로 지정하지 않아도 params 를 jQuery object 로 만들지 않기 때문에 array 타입으로 전송됩니다.

When Communicator is used as N.comm(params, url).submit();, params are not created as jQuery objects even if the dataIsArray option is not set to true, so they are sent as an array type.

Natural-ARCHITECTURE v0.8.1.4 버전 이후에 적용되었습니다.

Natural-ARCHITECTURE v0.8.1.4 or later.
target jQuery object null X

HTML 콘텐츠를 삽입할 요소를 지정합니다.

Specifies the element to insert HTML content.

Communicator 를 N(".block").comm("page.html").submit();와 같이 사용할 경우 N("#block") 요소 오브젝트가 target 속성 값으로 지정됩니다.

If Communicator is used as N(".block").comm("page.html").submit();, the N(".block") element object is assigned as the target attribute value.
contentType string application/json; charset=utf-8 O

서버 요청에 대한 contentType 을 지정합니다.

Specifies the contentType for server requests.

contentType 옵션은 jQuery.ajax 함수의 옵션이 그대로 적용됩니다. jQuery.ajax 매뉴얼의 settings 부분에서 contentType 속성을 참고하세요.

The contentType option remains the same as the jQuery.ajax function option. See the contentType property in the settings section of the jQuery.ajax manual.

cache boolean false X

true 로 설정하면 요청된 페이지가 브라우저에 의해 캐싱됩니다.

If set to true, it will force requested pages to be cached by the browser.

cache 옵션은 jQuery.ajax 함수의 옵션이 그대로 적용됩니다. jQuery.ajax 매뉴얼의 settings 부분에서 cache 속성을 참고하세요.

The cache option remains the same as the jQuery.ajax function option. See the cache property in the settings section of the jQuery.ajax manual.

type string POST X

요청에 사용할 HTTP method(예 : "POST", "GET", "PUT")

The HTTP method to use for the request (e.g. "POST", "GET", "PUT").

type 옵션은 jQuery.ajax 함수의 옵션이 그대로 적용됩니다. jQuery.ajax 매뉴얼의 settings 부분에서 type 속성을 참고하세요.

The type option remains the same as the jQuery.ajax function option. See the type property in the settings section of the jQuery.ajax manual.

data json object array1)|json object null X

서버로 전송될 데이터. string 이 아닌 경우 string 으로 변환됩니다.

Data to be sent to the server. It is converted to a string, if not already a string.

data 옵션은 jQuery.ajax 함수의 옵션이 그대로 적용됩니다. jQuery.ajax 매뉴얼의 settings 부분에서 data 속성을 참고하세요.

The data option remains the same as the jQuery.ajax function option. See the data property in the settings section of the jQuery.ajax manual.

dataType string(xml, json, script, or html) "json" X

서버 응답 데이터의 유형(xml, json, script, or html)을 설정합니다.

Set the server response data type(xml, json, script, or html).

dataType 옵션은 jQuery.ajax 함수의 옵션이 그대로 적용됩니다. jQuery.ajax 매뉴얼의 settings 부분에서 dataType 속성을 참고하세요.

The dataType option remains the same as the jQuery.ajax function option. See the dataType property in the settings section of the jQuery.ajax manual.

crossDomain boolean false X

동일한 도메인에서 crossDomain 요청 (예 : JSONP)을 강제로 수행하려면 crossDomain 값을 true 로 설정하세요. 이를 통해서 다른 도메인으로 서버 사이드 리다이렉션 하는 것 등이 가능해집니다.

If you wish to force a crossDomain request(such as JSONP) on the same domain, set the value of crossDomain to true. This allows, for example, server-side redirection to another domain.

crossDomain 옵션은 jQuery.ajax 함수의 옵션이 그대로 적용됩니다. jQuery.ajax 매뉴얼의 settings 부분에서 crossDomain 속성을 참고하세요.

The crossDomain option remains the same as the jQuery.ajax function option. See the crossDomain property in the settings section of the jQuery.ajax manual.

N.comm 은 jQuery.ajax 모듈을 사용하여 Ajax 요청을 처리합니다. jQuery.ajax 의 beforeSend, success, error, complete 옵션을 제외 한 옵션들은 N.comm 에서도 그대로 적용됩니다.

N.comm uses the jQuery.ajax module to process Ajax requests. The options in jQuery.ajax also apply to N.comm except for beforeSend, success, error and complete options.
