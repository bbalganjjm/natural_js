["date", 4|6|8|10|12|14, "month"|"date", { "datepicker options" : "" }]

날짜 형식으로 변환 합니다.

Converts to date format.

기본 날짜 형식은 Config(natural.config.js)의 N.context.attr("data").formatter.date 속성에서 설정 가능합니다.

The default date format can be set in N.context.attr("data").formatter.date property of Config(natural.config.js).

1. 다음은 두 번째 인자 값에 대한 설명입니다.

1. The following is the description of the second argument value.

number 타입으로 길이를 지정하면 다음과 같이 변환 됩니다.

If you specify the length as a number type, it is converted as follows.

  • 4 : 년
  • 6 : 년-월
  • 8 : 년-월-일
  • 10 : 년-월-일 시
  • 12 : 년-월-일 시:분
  • 14 : 년-월-일 시:분:초
  • 4: Year
  • 6: Year-Month
  • 8: Year-Month-Day
  • 10: Year-Month-Day Hour
  • 12: Year-Month-Day Hour:Minute
  • 14: Year-Month-Day Hour:Minute:Second

string 타입으로 데이트 포멧 룰을 지정하면 다음과 같이 변환 됩니다.

If you specify a date format rule as a string type, it is converted as follows.

  • Y : 년(4자리)
  • y : 년(2자리)
  • m : 월
  • d : 일
  • H : 시
  • i : 분
  • s : 초
  • Y: Year(4 digits)
  • y: year(2 digits)
  • m: month
  • d: day
  • H: Hour
  • i: minute
  • s: second
"1999/12/31" : "Y/m/d"
"99/12/31" : "y/m/d"
"31/12" : "d/m"
"12/31/1999" : "m/d/Y"
"1999-12-31 12:01:59" : "Y-m-d H:i:s"

2. 다음은 세 번째 인자 값에 대한 설명입니다.

2. The following is the description of the third argument value.

  • date : Datepicker가 표시 됩니다.
  • date : Datepicker is displayed.
  • month : Monthpicker가 표시 됩니다.
  • month : Monthpicker is displayed.

3. 네 번째 인자는 Datepicker의 옵션들을 JSON 형식의 문자열로 선언하면 컴포넌트의 옵션이 적용 됩니다.

3. In the fourth argument, if you declare Datepicker's options as a JSON-formatted string, the component's options apply.

["generic", "사용자서식"]

사용자 서식을 이용하여 문자열을 서식화 합니다.

Formats a string using a user format.

generic과 numeric 룰은 Mask JavaScript API (v0.4b) 라이브러리를 기본으로 개발 되었습니다. Date 마스킹 부분은 제거하고 한글인식 부분과 0보다 작은 값들(수수점 뒤 값들)에 대한 처리를 추가하고 * 문자와 공백 문자를 허용(기존 * 문자는 ~ 문자로 변환 됨) 하도록 기능을 변경 하였습니다. 자세한 사용법과 예는 위 링크를 참고하세요.

The generic and numeric rules were developed based on the Mask JavaScript API (v0.4b) library. The date masking part was removed, and the Hangul recognition part and processing for values ​​less than 0(values ​​after the decimal point) were added, and the function was changed to allow * characters and space characters(existing * character is converted to ~ character). For detailed usage and examples, please refer to the link above.

  • # : 숫자, 공백문자
  • @ : 한글(자음/모음), 영문자, 공백문자
  • ~ : 한글(자음/모음), 영문자, 숫자, 공백문자
  • # : Number, Space character
  • @ : Hangul(consonants/vowels), English characters, Space characters
  • ~ : Hangul(consonants/vowels), English characters, Number, Space characters

["numeric" "사용자서식", "option"]

사용자 서식을 이용하여 숫자 문자열을 서식화 합니다.

Formats a numeric string using a user format.

option 인자 값에 따라 소숫점을 다음과 같이 처리 합니다.

Depending on the value of the option argument, the decimal point is processed as follows.

  • ceil : 무조건 올림(소수점 처리가능)
  • floor : 무조건 버림(소수점 처리가능)
  • round : 반올림(소수점 처리가능)
  • ceil : Ceil(possible decimal point processing)
  • floor : Floor(possible decimal point processing)
  • round : Round(decimal point processing possible)

generic과 numeric 룰은 Mask JavaScript API (v0.4b) 라이브러리를 기본으로 개발 되었습니다. Date 마스킹 부분은 제거하고 한글인식 부분과 0보다 작은 값들(수수점 뒤 값들)에 대한 처리를 추가하고 * 문자와 공백 문자를 허용(기존 * 문자는 ~ 문자로 변환 됨) 하도록 기능을 변경 하였습니다. 자세한 사용법과 예는 위 링크를 참고하세요.

The generic and numeric rules were developed based on the Mask JavaScript API (v0.4b) library. The date masking part was removed, and the Hangul recognition part and processing for values ​​less than 0(values ​​after the decimal point) were added, and the function was changed to allow * characters and space characters(existing * character is converted to ~ character). For detailed usage and examples, please refer to the link above.
