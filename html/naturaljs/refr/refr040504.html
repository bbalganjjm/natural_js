기본옵션 Default Options

Name Type Default Required Description
context jQuery object|jquery selector string null O

N.tab을 적용할 context 요소를 지정 합니다.

Specifies the context element to apply N.tab.

N.tab의 context 요소는 반드시 div 태그 안에 ul, li 와 div 태그로 작성해야 합니다.

The context element of N.tab must be written with ul, li and div tag in div tag.

  • tab
    • tab 요소는 ul 태그의 li 태그를 사용 합니다.
    • The tab element uses the li tag of the ul tag.
    • tab 링크(a 태그) 태그의 href 속성값과 tab contents(div) 태그의 id 속성값을 맞춰 줍니다.
    • Match the href attribute value of the tab link(a tag) tag with the id attribute value of the tab contents(div) tag.
  • tab contents
    • tab contents 요소는 div 태그를 사용 합니다.
    • The tab contents element uses div tags.
    • tab(li) 의 순서와 개수에 맞춰 tab contents(div)를 생성 합니다.
    • Create tab contents(div) according to the order and number of tab(li).
    • tab contents(div) 태그의 id 속성과 tab 링크(a 태그)의 href 속성값을 맞춰 줍니다.
    • Match the id attribute of the tab contents(div) tag with the value of the href attribute of the tab link(a tag).
    • 탭 옵션에 url 옵션을 설정하여 다른 페이지를 불러올 수 있고 tab contents 요소(div) 안에 직접 컨텐츠를 작성 해도 됩니다.
    • You can load other pages by setting the url option in the tab options, and you can also write content directly inside the tab contents element(div).
<div class="tab-context">
    <ul>
        <li><a href="#tab1">Tab1</a></li>
        <li data-opts='{ "url" : "tab1.html" }'>
            <a href="#tab2">Tab2</a>
        </li>
        <li data-opts='{ "url" : "tab2.html"}'>
            <a href="#tab3">Tab3</a>
        </li>
    </ul>
    <div id="tab1">Tab1</div>
    <div id="tab2"></div>
    <div id="tab3"></div>
</div>
tabOpts array[object] [] X

탭 요소의 data-opts 속성이 아닌 array[object] 타입으로 개별 탭의 옵션을 지정 합니다.

Specify options for individual tabs by the array[object] type, not the data-opts attribute of the tab element.

옵션은 탭의 순서와 개수 만큼 옵션 object로 설정 합니다.

Option is set as option object as much as the order and number of tabs.

N("#tab").tab({
    tabOpts : [
        { width: "auto", url: "tab1.html", preload: false, active: false }, //Tab1
        { width: "auto", url: "tab2.html", preload: false, active: false }, //Tab2
        { width: "auto", url: "tab3.html", preload: false, active: false, onOpen: "onOpen" }  //Tab3
    ]
});

탭 태그(li)에 직접 data-opts 속성으로 옵션을 지정할 수도 있습니다.

You can also specify options with the data-opts attribute directly on the tab tag(li).

개별 옵션에 대한 설명은 [선언형 옵션] 탭을 참고 바랍니다.

For details on individual options, please refer to the [Declarative Options] tab.
tabScroll boolean false X

true로 설정하면 마우스 드래그나 터치 또는 처음 / 마지막 버튼으로 탭들을 스크롤할 수 있습니다.

If set to true, the tabs can be scrolled with mouse drag or touch or first / last button.

처음 / 마지막 버튼으로 탭들을 스크롤 하려면 ul 태그 처음과 마지막 자식 요소로 a 태그와 span 태그를 작성 해 주면 됩니다.

To scroll tabs with the first / last button, write a tag and span tag with the first and last child elements of the ul tag.

<div>
    <a href="#"><span></span></a> <!-- First button -->
    <ul>
        <li><a href="#tab1">tab1</a></li>
        <li><a href="#tab2">tab2</a></li>
        <li><a href="#tab3">tab3</a></li>
    </ul>
    <a href="#"><span></span></a> <!-- Last button -->
    <div id="tab1">tab1</div>
    <div id="tab2">tab2</div>
    <div id="tab3">tab3</div>
</div>
tabScrollCorrection boolean false X

탭 요소에 영향을 주는 스타일(CSS)에 의해 마지막 탭이 잘리거나 여백이 생길 수 있습니다. 이때 tabScrollCorrection 객체의 속성으로 다음 옵션 값들을 조절하여 정상적으로 표시되게할 수 있습니다.

The style (CSS) affecting the tab element may cause the last tab to be truncated or margins. At this time, you can adjust the following option values ​​as properties of the tabScrollCorrection object to display normally.

  • tabContainerWidthCorrectionPx : 마지막 탭이 잘리거나 여백이 생겼을 때 1씩 증감하여 탭의 모양을 보정할 수 있는 옵션입니다.
  • tabContainerWidthCorrectionPx : This option allows you to correct the shape of the tab by increasing or decreasing by 1 when the last tab is cut or a margin is created.
  • tabContainerWidthReCalcDelayTime : 탭이 처음 표시 될 때 탭이 잘리거나 여백이 생겼을 때 1씩 증감해서 탭의 모양을 보정할 수 있는 옵션입니다.
  • tabContainerWidthReCalcDelayTime : This is an option to correct the shape of the tab by incrementing by 1 when the tab is cut or a margin is created when the tab is first displayed.
N("#tab").tab({
    tabScrollCorrection : {
        tabContainerWidthCorrectionPx : 1,
        tabContainerWidthReCalcDelayTime : 0
    }
});

Config(natural.config.js) 의 N.context.attr("ui").tab 프로퍼티에 tabScrollCorrection 옵션을 설정하면 전체 N.tab 컴포넌트에 적용 됩니다.

Setting the tabScrollCorrection option in the N.context.attr("ui").tab property of Config(natural.config.js) applies to all N.tab components.
randomSel boolean false X

true로 설정하면 탭이 초기화 될 때 탭과 탭컨텐츠를 랜덤으로 보여줍니다. false로 설정하면 첫 번째 탭을 보여줍니다.

If set to true, the tab and tab contents are displayed randomly when the tab is initialized. If set to false, displayed the first tab.

탭 옵션 중 active 옵션이 true 로 설정 되어 있으면 active 옵션이 먼저 적용 됩니다.

If the active option among the tab options is set to true, the active option is applied first.
blockOnActiveWhenCreate boolean false

true로 설정하면 탭을 생성 되고 기본 탭이 선택할 때 onActive 이벤트가 실행 되지 않습니다.

When set to true, the onActive event does not fire when a tab is created and then the default tab is selected.
opener N.cont's object null X

탭 옵션에 url 옵션을 설정 하여 다른 페이지를 로드할 경우 탭 컨텐츠의 Controller(N.cont) object 에서 N.tab 인스턴스를 생성한 부모페이지의 Controller object를 참조하기 위한 옵션입니다.

If loading another page by setting the url option in the tab option, option to refer to the Controller object of the parent page that created the N.tab instance from the Controller(N.cont) object of the tab content.

N.tab 인스턴스 생성 시 opener 옵션에 Controller object를 지정하면 탭 컨텐츠 페이지의 Controller object의 opener 속성으로 전달 됩니다.

When creating a N.tab instance, if a Controller object is specified in the opener option, it is passed as the opener property of the Controller object of the tab content page.

effect

Deprecated.

 boolean|array false

탭 컨텐츠가 표시될때 전환 효과를 보여줄지 여부를 설정 합니다.

효과를 적용하지 않으려면 false를 입력하고 효과를 입력하려면 다음과 같이 입력합니다.

["이펙트", "시간(ms)", "이펙트가 끝날 때 마다 실행 될 콜백함수"]

N("#tab").tab({
    effect : ["slideDown", 300, null]
});

이펙트 종류 : show, hide, slideUp, slideDown, fadeIn, fadeOut
onActive function null X

탭이 활성화 될 때마다 실행되는 이벤트 핸들러를 정의 합니다.

Define an event handler that is executed whenever the tab is activated.

onActive : function(tabIdx, tabEle, contentEle, tabEles, contentEles) {
    // this : N.tab 인스턴스
    // tabIdx : 활성화된 탭의 Index
    // tabEle : 활성화된 탭의 요소
    // contentEle : 활성화된 탭의 컨텐츠 요소
    // tabEles : 전체 탭 요소
    // contentEles : 전체 컨텐츠 요소
}
onActive : function(tabIdx, tabEle, contentEle, tabEles, contentEles) {
    // this : N.tab instance
    // tabIdx : Index of active tab
    // tabEle : Element of active tab
    // contentEle : Content element of active tab
    // tabEles : Entire tab element
    // contentEles : Entire content elements
}

탭 컨텐츠가 처음 로딩 될 경우 Controller object의 init 함수가 실행 된 다음 onActive 함수가 실행 됩니다.

If the tab content is loaded for the first time, the init function of the Controller object is executed, and then the onActive function is executed.

Defines an event handler that is executed when tab content loading is completed.

onLoad : function(tabIdx, tabEle, contentEle, cont) {
    // this : N.tab 인스턴스

    // cont : 로딩 된 탭 컨텐츠의 Controller object
}
onLoad : function(tabIdx, tabEle, contentEle, cont) {
    // this : N.tab instance
    // tabIdx : Index of active tab
    // tabEle : Element of active tab
    // contentEle : Content element of active tab
    // cont : Controller object of the loaded tab content
}
