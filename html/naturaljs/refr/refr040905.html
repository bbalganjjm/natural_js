선언형 옵션 Declarative Options

N.grid의 선언형 옵션은 요소의 data-* 속성에 설정하는 옵션입니다.

The declarative option of N.grid is an option set in the data-* attribute of an element.

...
<span id="inputDate" data-format='[ ["date", 8] ]'></span>
<input id="inputDate" type="text" data-validate='[ ["required"], ["integer"] ]' data-format='[ ["trimToEmpty"], ["date", 8, "date"] ]'>
...
<th data-filter="true"></th>
<th data-rowspan="true"></th>
...

선언형 옵션은 JSON 표준 형식을 정확하게 준수해야 합니다(프로퍼티 명은 반드시 큰 따옴표로 감싸야합니다). JSON 표준 포맷을 준수하지 않으면 선언형 옵션이 인식되지 않거나 오류가 발생합니다.

Declarative options must conform exactly to the JSON standard format(Property names must be enclosed in double quotes). If you do not comply with the JSON standard format, the declarative options are not recognized or an error occurs.

선언형 옵션에 대한 설명은 API 문서 안내 메뉴의 선언형 옵션 섹션을 참고하세요.

For a description of the declarative options, see the Declarative Options section of the API Documentation Guide menu.

data-filter string undefined

filter 기본 옵션 값을 false로 설정하고 table thead의 th에 data-filter="true" 속성을 정의하면 해당 컬럼만 필터 기능이 활성화됩니다.

If you set the filter default option value to false and define the data-filter = "true" attribute in th of table thead, the filter function is activated only for the corresponding column.

필터링될 컬럼의 정보는 tbody의 td 나 td 안에 있는 요소들의 id 값을 기준으로 필터링합니다. 그러나 tbody의 td 안에 id 속성을 가진 요소가 2개 이상 존재하거나 그리드 헤더와 바디의 요소 구성이 많이 다를 경우에는 제대로 필터링이 되지 않을 수 있습니다. 이때는 th 요소의 data-id 속성에 필터링될 컬럼명을 선언해 주면 정상적으로 작동됩니다.

The information in the column to filter is filtered based on the id value of the elements in td or td of tbody. However, if there are two or more elements with the id attribute in td of tbody, or if the element construction of the grid header and body is very different, it may not be properly filtered. In this case, if the column name to be filtered is declared in the data-id attribute of th element, it works normally.

data-rowspan string undefined

열(컬럼)을 기준으로 같은 값이 포함되어 있는 셀 들을 병합합니다.

Merge cells that contain the same value based on the column.

data-rowspan="true" 속성을 병합할 열(th 태그)에 추가합니다.

Adds the data-rowspan="true" attribute to the columns to be merged(th tag).

  • 물리적으로 요소들이 병합되지 않고 병합된 것처럼 보이도록 처리합니다.
  • 이전 열의 병합이 풀리면 해당 열의 값이 같더라도 병합이 풀리고 다음 행에서 새로운 병합이 시작됩니다.
  • 병합되는 셀에 입력 요소가 있으면 병합된 열 그룹의 첫 번째 행 안에 있는 입력 요소만 컴포넌트의 data와 동기화됩니다(병합된 나머지 셀의 입력 요소는 제거됩니다).
  • 병합될 컬럼의 정보는 tbody의 td 나 td 안에 있는 요소들의 id 값을 기준으로 병합합니다. 그러나 tbody의 td 안에 id 속성을 가진 요소가 2개 이상 존재하거나 그리드 헤더와 바디의 요소 구성이 많이 다를 경우에는 제대로 병합이 되지 않을 수 있습니다. 이때는 th 요소의 data-id 속성에 병합될 컬럼명을 선언해 주면 정상적으로 작동됩니다.
  • It works so that the elements are not physically merged but appear to be merged.
  • When the previous column is unmerged, the merge is unmerged even if the values in that column are the same, and a new merge is started in the next row.
  • If there are input elements in the merged cell, only the input elements in the first row of the merged column group are synchronized with the component's data(The input elements of the remaining merged cells are removed).
  • The information in the column to merge is merged based on the id value of the elements in td or td of tbody. However, if there are two or more elements with the id attribute in td of tbody, or if the element construction of the grid header and body is very different, it may not be properly merged. In this case, if the column name to be merged is declared in the data-id attribute of th element, it works normally.
