예제 Examples

1. N.form이나 N.grid로 데이터를 바인딩하기 전에, 컬럼 값이 바인딩될 요소의 data-format 속성에 포맷 규칙을 선언하기.

1. Before binding data with N.form or N.grid, declare format rules in the data-format attribute of the element where the column value will be bound.

<input id="startDate" type="text" data-format='[["date", 8]]'>

위와 같이 HTML 요소에 선언하면, 포커스가 해당 요소에서 벗어날 때 포맷팅이 적용되고, 다시 요소에 포커스되면 포맷팅이 해제됩니다.

When declared on an HTML element as described above, formatting is applied when the focus leaves the element and is removed when the element is focused again.

<span id="startDate" data-format='[["date", 8]]'></span>

위와 같이 입력 요소가 아닌 요소에 포맷 규칙을 선언하면, 포맷된 값이 요소의 텍스트로 입력됩니다.

When format rules are declared on a non-input element as described above, the formatted value is set as the element's text.

Formatter는 N.grid나 N.form등의 UI 컴포넌트에 입력된 원 데이터를 포맷팅하여 요소에 바인딩하지 않습니다. 원 데이터는 그대로 유지한 채 화면에 보이는 값들만 포맷팅하기 때문에 서버에 데이터를 전송할 때 다시 unformat하지 않아도 됩니다.

The Formatter does not bind formatted values to elements in UI components like N.grid or N.form; it only formats the displayed values while keeping the original input data unchanged. Therefore, there is no need to unformat the data when sending it to the server.

2. N.formatter 라이브러리를 사용하여 포맷팅하기.

2. Formatting using the N.formatter library.

입력 요소에 data-format='[["limit", 12, "..."]]' 와 같이 포멧 규칙을 선언하면 프레임워크 내부에서는 "limit" 다음의 12 와 "..." 을 [12, "..."]와 같이 Array 형태로 변환하여 N.formatter.limit 함수의 인수로 실행합니다.

When declaring a format rule on an element, in data-format='[["limit", 12, "..."]]' attribute value, "date" is the formatting rule, and after 8 is the format parameters, and the string to be formatted is the value of the input element or the text of other element.

포맷 규칙 함수를 명령으로 처리하는 방법은 다음과 같습니다.

The method for handling format rule functions as commands is as follows.

// N.formatter.{rule}("String to format", ...arguments);

N.formatter.limit("abcdefghijklmn", [12, "..."]); // 'abcdefghijkl...'

3. 요소에 바인딩된 Formatter 관련 이벤트를 직접 실행하기.

3. Directly triggering formatter-related events bound to an element.

Formatter는 룰 대신 입력 요소를 지정하면, 입력 요소에 "format" 이벤트와 "unformat" 이벤트를 생성합니다. 이 이벤트들은 jQuery의 trigger 메서드를 통해 직접 실행할 수 있습니다.

When you specify an input element instead of a rule for the Formatter, it generates "format" and "unformat" events on the input element. These events can be directly triggered using jQuery's trigger method.

N("input[name='targetEle']").trigger("unformat");

N("input[name='targetEle']").trigger("format");

4. 동적으로 포맷 규칙을 변경하고 즉시 반영하기.

4. Dynamically change and immediately apply format rules.

N("selector").data("format", [["trimtoempty"]]).trigger("format");

위 코드가 작동하기 위해서는 컴포넌트로 데이터를 바인딩하기 전에 대상 요소에 data-format 속성으로 포멧 규칙이 1개 이상 선언되어 있어야 합니다.

For the code to function properly, at least one format rule must be declared in the data-format attribute of the target element before binding data to the component.

전체 포맷 규칙에 대한 설명은 [포맷 룰 목록] 탭을 참고하세요.

Refer to the [Format rule list] tab for a full description of the format rules.
