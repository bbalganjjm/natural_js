data array[json object] undefined O

N.pagination에 바인딩할 데이터를 지정합니다.

Specifies the data to bind to N.pagination.

data옵션이 지정되면 totalCount 값이 자동으로 계산되어 설정되므로 totalCount 값을 0으로 설정하거나 설정하지 않아야 합니다.

If the data option is specified, the totalCount value is automatically calculated and set, so the totalCount value must be set to 0 or not.

DB 페이징을 하는 경우 data 옵션을 지정하지 말고 totalCount 값만 서버에서 가져와 설정하기 바랍니다.

In case of DB paging, do not specify the data option but get the totalCount value from the server and set it.
context jQuery object|jquery selector string null O

N.pagination을 적용할 context 요소를 지정합니다.

Specifies the context element to apply N.pagination.

N.pagination의 context 요소는 반드시 div 태그 안에 ul, li 태그로 작성해야 합니다.

The context element of N.pagination must be written with ul, li tag in div tag.

  • 처음 페이지, 마지막 페이지, 이전 페이지, 다음 페이지 First page, Last page, Previous page, Next page
    • ul 태그의 li 태그에 a 태그를 넣어서 작성합니다.
    • Write "a" tag by inserting "a" tag in li tag of ul tag.
    • 처음 페이지와 이전 페이지 요소, 마지막 페이지와 다음 페이지 요소는 ul 태그를 분리하여 작성합니다.
    • Write the first and previous page elements and the last and next page elements in separate ul tags.
    • 처음 페이지, 마지막 페이지 링크와 관련된 태그들을 작성하지 않으면 관련 기능이 비활성화됩니다.
    • If you do not write tags related to the first page or last page link, the related feature is disabled.
  • 페이지 인덱스 Page index
    • ul 태그의 li 태그에 a 태그를 넣어서 작성합니다.
    • Write by inserting a tag in li tag of ul tag.
<div class="pagination-context">
    <ul>
        <li><a href="#">first</a></li>
        <li><a href="#">prev</a></li>
    </ul>
    <ul>
        <li><a href="#"><span>1</span></a></li>
    </ul>
    <ul>
        <li><a href="#">next</a></li>
        <li><a href="#">last</a></li>
    </ul>
</div>

currPageNavInfo 객체 정보

currPageNavInfo object information

  • pageNo : 현재 페이지 번호.
  • pageNo : Current page numbe.r
  • countPerPage : 페이지 당 행 수
  • countPerPage : Rows count per page.
  • countPerPageSet : 페이지 세트당 페이지 수.
  • countPerPageSet : Pages count per page set.
  • currSelPageSet : 현재 페이지 세트 번호.
  • currSelPageSet : Current page set number.
  • pageCount : 전체 페이지 수.
  • pageCount : Total pages count.
  • pageSetCount : 전체 페이지 세트 수.
  • pageSetCount : Total page sets count.
  • totalCount : 전체 행 수.
  • totalCount : Total rows count.
  • startPage : 현재 페이지 세트 중 첫 번째 페이지 번호.
  • startPage : Number of the first page in the current page set.
  • startRowIndex : 선택한 페이지의 첫 번째 행 인덱스.
  • startRowIndex : Index of the first row on the selected page.
  • startRowNum : 선택한 페이지의 첫 번째 행 번호
  • startRowNum : Number of the first row of the selected page.
  • endPage : 현재 페이지 세트 중 마지막 페이지 번호.
  • endPage : Number of the last page in the current page set.
  • endRowIndex : 선택한 페이지의 마지막 행 인덱스.
  • endRowIndex : Index of the last row on the selected page.
  • endRowNum : 선택한 페이지의 마지막 행 번호.
  • endRowNum : Number of the last row of the selected page.

DB 페이징을 하는 경우 서버에서 totalCount 값을 먼저 가져온 다음 onChange 이벤트 핸들러의 currPageNavInfo 인수 값을 서버에 파라미터로 전송하여 페이징 된 데이터를 조회할 수 있습니다.

In case of DB paging, get the totalCount value from the server first. and then you can retrive the paged data by sending the value of the currPageNavInfo argument of the onChange event handler as a parameter to the server.

이 페이징 된 데이터를 onChange 이벤트에서 N.grid 나 N.list에 페이지가 바뀔 때마다 바인딩해 주면 됩니다.

Bind this paged data to N.grid or N.list in onChange event whenever the page changes.

var grid = N(data).grid(".grid-context");

    N.comm("getTotalCnt.json").submit(function(data){
        N(data).pagination({
            context: ".pagination-context",
            totalCount: data.totalCount,
            onChange: function(pageNo, selEle, selData, currPageNavInfo) {
                N(currPageNavInfo).comm("getPagedDataList.json").submit(function(data){
                    grid.bind(selData);
                });
            }
        }).bind();
    });

blockOnChangeWhenBind 옵션이 true로 설정되어있을 경우 페이징 버튼들을 클릭할 때만 이벤트가 실행됩니다.

If the blockOnChangeWhenBind option is set to true, the event is executed only when the paging buttons are clicked.
